# Disclaimer — authorized testing only

HackingPal is a security testing workspace. It bundles tools — port
scanners, vulnerability probes, web-application attack modules,
credential testers, network capture, cloud and Active Directory
enumeration — that are useful in legitimate security work and that
are illegal to point at infrastructure you do not own or have
written authorization to test.

1. **You will only use it against systems you own, or against systems
   for which you have explicit written authorization to perform
   security testing** (a signed Rules of Engagement, a pentest
   contract, a bug-bounty program's in-scope target list, a CTF
   environment, a lab you control).
2. **Authorization is your responsibility.** The application's
   "Lab" vs "Engagement" mode toggle, scope checks, and per-page
   authorization checkboxes are operator-assist features. They are
   not, and cannot be, proof that you have permission. Misuse is
   on the operator, not the tooling.
3. **You will follow all applicable laws.** Unauthorized access,
   unauthorized scanning, exceeding authorized scope, and
   unauthorized data exfiltration are crimes in most jurisdictions
   (e.g. the US Computer Fraud and Abuse Act, the UK Computer
   Misuse Act, equivalents elsewhere). Bug-bounty safe harbour
   covers only the program's listed scope and rules.
4. **You will respect the engagement.** Stay inside the scope you
   were granted. Stop when asked. Disclose findings through the
   channel agreed with the asset owner, not publicly.
